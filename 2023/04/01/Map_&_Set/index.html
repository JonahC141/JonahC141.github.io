Map

Map是一种新的集合类型，为这门语言带来了真正的键/值存储机制！

​ Map是一组键值对的结构，具有极快的查找速度。

Map对象常用方法：

Map.set() ——为Map对象添加或更新一个指定了key和value的键值对。

Map.delete()——用于删除Map对象中指定的元素。

Map.has()——方法返回一个布尔值，判断具有指定键的元素是否存在。

Map.get()——方法从Map对象返回指定的元素。如果与所提供的键相关联的值是一个对象，那么你将获得该对象的引用，对该对象所做的任何更改都会有效地在 Map 对象中修改它。

Map.clear()——清空Map对象中的所有元素。

Map.forEach()——与数组的forEach()相似

Map.keys()——返回一个引用的迭代器对象，包含按照顺序插入Map对象的中每个元素的key值

Map.values()——方法返回一个新的迭代器对象。它包含按顺序插入 Map 对象中每个元素的 value 值。

Map.entries()——方法返回一个新的迭代器对象，其中包含 Map 对象中按插入顺序排列的每个元素的 [key, value] 对。在这种情况下，这个迭代器对象也是一个可迭代对象，因此可以使用 for-of 循环。当使用 [Symbol.iterator] 时，它返回一个函数，该函数在调用时返回迭代器本身。

关于Map
  1. 由于一个key只能对应一个value，所以，多次对一个key放入value，后面的值会把前面的值覆盖。
  2. 可以通过Map.size确定map实例的长度。
  3. 可以通过Symbol.itertor生成[key,value]形式的数组，它引用entries()。
  4. 选择Object还是Map？
    • Map大约可以比Object多存储50%的键值对。
    • 如果代码涉及大量插入操作，那么显然Map的性能更佳。
    • 大量查找操作，那么某些情况下Object更好。
    • 涉及大量删除操作，选择Map。

Set

​ ES6新增的Set是一种新集合类型，为这门语言带来集合数据结构。Set在很多方面都像是加强的Map。

Set对象允许你储存任何类型的唯一值，无论是原始值或者是对象引用。

Set对象是值的集合，其中的元素是唯一的。

特别地，NaN 和 undefined 都可以被存储在 Set 中，NaN 之间被视为相同的值（NaN 被认为是相同的，尽管 NaN !== NaN）。

Set.add()——将插入一个具有指定值的新元素到 Set 对象中。

Set.delete()——从 Set 对象中删除指定的值（如果该值在 Set 中）。

​ delete()返回一个布尔值，表示集合中是否存在要删除的值

Set.has()——返回一个布尔值来指示对应的值是否存在于 Set 对象中。

Set.clear()——移除 Set 对象中所有元素。

Set.forEach()——和数组的forEach方法很相似，只不过Set结构的键名就是键值，所以第一个参数与第二个参数的值永远都是一样的。

Set.keys()——同values()

Set.values()——同keys()

Set.entries()——返回一个新的迭代器对象，最终得到一个[value,value]形式的数组

这部分跟Map很相似，不必多说

关于Set
  1. Set可以用于数组去重
  2. Set集合是大小写敏感的
